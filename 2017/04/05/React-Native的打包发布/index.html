React Native 的调试与打包发布

Android手机要求所有应用都有一个数字签名才会被允许安装在用户手机上，发布到各个平台。

  1. 生成一个签名密钥（通过命令的方式）,进入到你的项目目录下：

生成一个叫做my-release-key.keystore的密钥库文件。
密钥库里应该已经生成了一个单独的密钥，有效期为10000天。–alias参数后面的别名是你将来为应用签名时所需要用到的，所以记得记录这个别名。

  1. 在/android…/main文件下创建一个assets文件夹。
  2. 在工程目录下将index.android.bundle文件下载并保存到assets文件夹下。通过这个命令

  1. 配置android\app文件加下build.gradle的android keystre的配置。
  1. 启用代码的混淆。
  1. 在/android/目录中执行gradle assembleRelease 命令，打包后android/app/build/outputs/apk文件夹下可以看到。首先要先下载gradle,在环境变量中配置，在执行上面命令。（注意gradle的版本要一致，在wrapper中的一致，在gradle-wrapper.properties查看gradle版本）
